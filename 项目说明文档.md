## 一、项目背景

Nexus Inspiration 是一个**设计灵感收集与管理平台**，供团队成员协作上传、分类、浏览和管理设计灵感卡片。

### 核心需求
- 多用户登录，各自上传灵感，所有人可查看全部内容
- 支持图片、视频、PDF、网页链接等多种附件类型
- 两级分类体系：一级分类（政策/经济/社会/技术/设计灵感），二级分类（仅设计灵感下可选）
- 列表/网格两种浏览模式，支持搜索和筛选
- 部署在公司内网服务器上，团队通过浏览器访问

| 字段 | 类型 | 说明 |

| id | UUID | 主键，自动生成 |

## 四、功能说明

### 新建灵感（/capture）
1. 选择一级分类（5选1）
2. 若选择「设计灵感」，需选择二级分类（必选）
3. 填写标题、描述（必填）
4. 选择信息来源（标签 + 补充文本，选填）
5. 填写设计启示（必填）
6. 添加自定义标签
7. 上传附件：支持**图片、视频、PDF**（单文件最大 120MB），也可粘贴网页链接

### 仪表盘（/dashboard）
- **浏览模式**：列表视图 / 网格视图切换
- **筛选**：一级分类筛选（显示数量），选择设计灵感时显示二级筛选
- **搜索**：按标题、分类、标签模糊搜索
- **详情面板**：点击卡片查看完整信息，包括附件预览、信息来源、设计启示
- **编辑**：点击编辑按钮可修改所有字段（仅本人上传的可编辑）
- **删除**：仅本人上传的可删除
- **显示信息**：每张卡片显示上传人和上传时间

## 七、数据库迁移说明

`db.ts` 中的 `initDatabase()` 在每次应用启动时自动执行，包含以下幂等迁移：

1. 创建 `inspirations` 表（如不存在）
2. 添加 `subcategory` 列（兼容旧表）
3. 英文分类映射中文：Policy→政策，Economy→经济，Sustainability→社会，Technology→技术
4. 平面→其他（二级分类更名）
5. 非设计灵感的行 subcategory 置 NULL
6. 无效分类的旧数据迁移为 设计灵感/其他
7. 添加 `source`、`source_text`、`design_insight` 列

所有迁移可重复执行，不会重复修改数据。

## 八、常见问题

**Q: 服务器重启后服务会自动恢复吗？**
A: 会。docker-compose 配置了 `restart: unless-stopped`，Docker 服务启动后容器自动恢复。

**Q: 上传的文件会丢失吗？**
A: 不会。文件存储在 Docker Volume 中，即使容器重建也会保留。路径：`/var/lib/docker/volumes/nexus-inspiration_nexus-uploads/_data/`

**Q: 如何添加新用户？**
A: 修改 `src/lib/auth.ts` 中的 `DEFAULT_USERS` 数组，添加新用户后重新部署。

**Q: 内网外的人能访问吗？**
A: 目前只能在 172.20.x.x 同网段内访问。如需外网访问，需要配置公网映射或 VPN。

**Q: Docker 镜像拉取超时怎么办？**
A: 服务器已配置国内镜像源（`/etc/docker/daemon.json`），如仍超时可尝试更换镜像源。
